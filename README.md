# High-Frequency Event-Driven Backtesting Engine

An ultra-low-latency, event-driven quantitative trading backtester built in C++17, featuring a Python-based ETL data pipeline and risk analytics dashboard. 

This engine is designed to simulate high-frequency trading (HFT) and statistical arbitrage strategies with realistic market friction (slippage and commissions). It processes tick data in **sub-microsecond** latency (~725 nanoseconds per tick) by utilizing $O(1)$ stateful indicators and fixed-point arithmetic for portfolio accounting.

## 🚀 Architecture

The system is decoupled into three distinct layers to maximize execution speed:

1. **ETL Data Pipeline (Python):** Ingests raw multi-asset market data from `yfinance`, cleans the data, forces chronological uniformity, and exports headerless CSVs to prevent branch-prediction misses in the C++ hot loop.
2. **Execution Engine (C++):** An event-driven state machine. It parses the pristine CSV, routes `MarketEvent`, `SignalEvent`, `OrderEvent`, and `FillEvent` objects, updates $O(1)$ technical indicators, and maintains a multi-asset `Portfolio` ledger.
3. **Analytics & Risk (Python):** Parses the output `trades.csv` generated by the C++ engine to calculate institutional risk metrics (Sharpe Ratio, Maximum Drawdown) and render a Matplotlib equity curve.

## ⚙️ Features

* **Multi-Asset Routing:** Tracks state and accounting for a universe of interleaved symbols simultaneously via `std::unordered_map` referencing.
* **$O(1)$ Technical Indicators:** Custom-built indicator library (SMA, EMA, RSI, Bollinger Bands) designed with rolling states to prevent $O(N)$ loop lookbacks.
* **Fixed-Point Accounting:** Protects portfolio cash balances from floating-point drift.
* **Realistic Execution Friction:** Simulates BPS-based slippage and flat-fee commissions.
* **Hardware Profiling:** Built-in `std::chrono` high-resolution timers to profile the hot path.

## 🚦 Quick Start Guide

### 1. Setup the Python Environment
Create and activate a virtual environment, then install the required dependencies.
```bash
python3 -m venv venv
source venv/bin/activate
pip install pandas yfinance matplotlib numpy
```

### 2. Run the Data Downloader (ETL)
Pull the historical market data. By default, this downloads 60 days of 5-minute interval data and formats it for the C++ engine.
```bash
python download_data.py
```
*Expected Output: `historical_data.csv` (No headers, clean timeline).*

### 3. Compile the C++ Engine
Compile the engine with the highest level of compiler optimization (`-O3`) to ensure sub-microsecond tick processing.
```bash
g++ -O3 -std=c++17 *.cpp -o backtester
```

### 4. Execute the Backtest
Run the compiled C++ binary. The engine will ingest the market data, execute the strategy, and output the trades.
```bash
./backtester
```
*Expected Output: Terminal printout of hardware execution speed and a `trades.csv` file containing the portfolio ledger.*

### 5. Generate Risk Analytics
Run the visualization script to calculate the Sharpe Ratio, Maximum Drawdown, and generate the equity curve dashboard.
```bash
python visualize.py
```
*Expected Output: Terminal printout of risk metrics and a `backtest_report.png` chart.*

## 📂 Project Structure

```text
├── DataHandler.h/cpp      # Ingests CSV and generates MarketEvents
├── Strategy.h/cpp         # Contains alpha logic and routes SignalEvents
├── Portfolio.h/cpp        # Multi-asset accounting and routing OrderEvents
├── ExecutionHandler.h/cpp # Simulates exchange matching and friction
├── Indicators.h           # O(1) SMA, EMA, RSI, and Bollinger Bands
├── Event.h                # Polymorphic event structures
├── BacktestEngine.h/cpp   # The main event loop and queue manager
├── download_data.py       # Python ETL script
├── visualize.py           # Python Risk Metrics & Matplotlib script
└── main.cpp               # Engine initialization and hardware profiling
```
